# Prometheus + Node Exporter 모니터링 구성 가이드

## 목표

DevStack Host에 Prometheus를 설치하고, OpenStack VM들에 Node Exporter를 설치하여 VM 모니터링 시스템을 구축함. GitLab Runner를 통한 Ansible 자동화로 전체 과정을 진행함.

## 아키텍처

```
[DevStack Host]
  Prometheus :9091  ──────────→  (:9100) Node Exporter [OpenStack VM #1]
                               └→ (:9100) Node Exporter [OpenStack VM #2...]
```

- GitLab 내장 Prometheus가 9090 포트를 사용하므로 우리 Prometheus는 9091 포트 사용
- 각 VM의 Node Exporter는 9100 포트에서 메트릭 노출

## DevStack Host Prometheus 설치

### 패키지 설치

```bash
sudo apt update
sudo apt install -y prometheus
prometheus --version
```

### 설정 파일 생성

```bash
sudo mkdir -p /etc/prometheus /var/lib/prometheus
sudo tee /etc/prometheus/prometheus.yml >/dev/null <<'YML'
global:
  scrape_interval: 15s

scrape_configs:
  # Prometheus 자기 자신
  - job_name: 'prometheus'
    static_configs:
      - targets: ['localhost:9091']

  # OpenStack VM(Node Exporter)
  - job_name: 'nodes'
    static_configs:
      - targets:
          - '221.157.174.34:9100'     # 필요 시 계속 추가
YML
sudo chown -R prometheus:prometheus /etc/prometheus /var/lib/prometheus
```

### 포트 변경 설정 (9091로 바인딩)

```bash
# systemd override로 ExecStart 고정
sudo mkdir -p /etc/systemd/system/prometheus.service.d
sudo tee /etc/systemd/system/prometheus.service.d/override.conf >/dev/null <<'EOF'
[Service]
EnvironmentFile=
ExecStart=
ExecStart=/usr/bin/prometheus \
  --config.file=/etc/prometheus/prometheus.yml \
  --storage.tsdb.path=/var/lib/prometheus \
  --web.listen-address=0.0.0.0:9091 \
  --web.console.templates=/usr/share/prometheus/consoles \
  --web.console.libraries=/usr/share/prometheus/console_libraries
EOF

sudo systemctl daemon-reload
sudo systemctl enable --now prometheus
sudo systemctl status prometheus --no-pager
sudo ss -lntp | grep :9091      # 0.0.0.0:9091 리슨 확인
curl -s http://localhost:9091/-/ready
```

## Node Exporter 자동 설치

### Ansible 플레이북 생성

`ansible/node-exporter.yml`

```yaml
---
- name: Install Node Exporter (simple version)
  hosts: all
  become: yes
  vars:
    ansible_user: ubuntu
    ansible_python_interpreter: /usr/bin/python3
    node_exporter_version: "1.8.2"

  tasks:
    - name: Add node_exporter user
      user:
        name: nodeexp
        system: yes
        shell: /usr/sbin/nologin
        create_home: no

    - name: Download node_exporter
      get_url:
        url: "https://github.com/prometheus/node_exporter/releases/download/v{{ node_exporter_version }}/node_exporter-{{ node_exporter_version }}.linux-amd64.tar.gz"
        dest: /tmp/node_exporter.tar.gz
        mode: "0644"

    - name: Extract node_exporter
      unarchive:
        src: /tmp/node_exporter.tar.gz
        dest: /usr/local/bin/
        remote_src: yes
        extra_opts: ["--strip-components=1"]

    - name: Create systemd service
      copy:
        dest: /etc/systemd/system/node_exporter.service
        content: |
          [Unit]
          Description=Prometheus Node Exporter
          After=network-online.target

          [Service]
          User=nodeexp
          ExecStart=/usr/local/bin/node_exporter --web.listen-address=":9100"
          Restart=always

          [Install]
          WantedBy=multi-user.target
      notify: Restart node_exporter

  handlers:
    - name: Restart node_exporter
      systemd:
        name: node_exporter
        state: restarted
        enabled: yes
```

### GitLab CI 파이프라인 수정

`gitlab_ci/terraform.ci.yml`의 `tf:ansible` 잡에서 기존 플레이북들 뒤에 추가:

```bash
# 4) Node Exporter 설치/기동
ansible-playbook -i "${TARGET_IP}," -u ubuntu --private-key ~/.ssh/mykey \
  --ssh-extra-args="-o StrictHostKeyChecking=no" ansible/node-exporter.yml
```

## 보안그룹 설정

### OpenStack 보안그룹 규칙 추가

VM에서 Node Exporter(9100 포트) 접근을 허용해야 함:

**실습용:**
```bash
# 모든 IP에서 접근 허용
remote_ip_prefix = "0.0.0.0/0"
```

**운영용 권장:**
```bash
# Prometheus 호스트 IP로 제한
remote_ip_prefix = "<DevStack_Host_IP>/32"
```

### Terraform 예시

```hcl
resource "openstack_networking_secgroup_rule_v2" "allow_node_exporter" {
  direction         = "ingress"
  ethertype         = "IPv4"
  protocol          = "tcp"
  port_range_min    = 9100
  port_range_max    = 9100
  remote_ip_prefix  = "0.0.0.0/0"   # 운영은 Prometheus 호스트 IP/32로 제한
  security_group_id = openstack_networking_secgroup_v2.monitor_sg.id
}
```

## 검증 체크리스트

### VM 내부 확인
```bash
# Node Exporter 메트릭 노출 확인
curl -s http://localhost:9100/metrics | head
```

### DevStack Host에서 원격 확인
```bash
# VM Node Exporter 접근 확인
curl -s http://<VM_IP>:9100/metrics | head
```

### Prometheus UI 확인
브라우저에서 `http://<DevStack_Host_IP>:9091/targets` 접속하여 확인:
- `prometheus` (localhost:9091) = UP
- `nodes` (<VM_IP>:9100) = UP

## 자주 사용하는 PromQL 쿼리

### 타겟 상태 확인
```promql
up
```

### CPU 사용률 (%)
```promql
100 - (avg by (instance) (rate(node_cpu_seconds_total{job="nodes",mode="idle"}[5m])) * 100)
```

### 메모리 사용률 (%)
```promql
(1 - node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes) * 100
```

### 루트 디스크 사용률 (%)
```promql
(1 - (node_filesystem_avail_bytes{mountpoint="/"} / node_filesystem_size_bytes{mountpoint="/"})) * 100
```

### 네트워크 트래픽 (B/s)
```promql
rate(node_network_receive_bytes_total[5m])
rate(node_network_transmit_bytes_total[5m])
```
